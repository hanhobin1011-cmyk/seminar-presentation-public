02. AI HANDLING

같은 문제라도
AI에게 주는 구조에 따라 결과가 달라집니다.

예시 문제

손풀이 제작 AI

함수 f(x)=x^3-3x^2+2의 극값을 구하고, 그래프 개형을 설명하시오.

원하는 결과
  • 핵심 아이디어가 먼저 보인다.
  • 미분과 부호 판단 과정이 빠지지 않는다.
  • 학생이 자주 하는 실수와 검산 항목이 들어간다.
  • 수업 준비용 손풀이 흐름으로 정리된다.
PROMPT ENGINEERING

프롬프트 엔지니어링은
한 번의 요청을 잘 쓰는 기술입니다.

너는 고등학교 수학 강사야. 아래 문제를 학생용 손풀이 해설로 작성해줘. 출력 형식: 1. 핵심 아이디어 2. 미분과 부호 판단 3. 그래프 개형 설명 4. 학생이 자주 하는 실수 5. 최종 답
장점빠르게 해설 초안이 나온다.
한계학생 수준과 강사 스타일이 매번 흔들릴 수 있다.

좋은 질문은 좋은 초안을 만들지만, 반복 가능한 시스템은 아직 아닙니다.

CONTEXT ENGINEERING

컨텍스트 엔지니어링은
AI가 내 수업을 이해하게 만드는 기술입니다.

수업 컨텍스트: - 대상: 고2 중위권 학생 - 현재 진도: 극대/극소 개념 도입 - 수업 목표: f'(x)=0보다 부호 변화 강조 - 자주 하는 실수: 모든 정지점을 극값으로 판단 - 해설 톤: 시험 직전 손풀이 노트처럼 간결하게
장점내 학생과 수업 흐름에 맞는 해설이 나온다.
한계컨텍스트가 부정확하면 결과도 그 방향으로 치우친다.

좋은 배경을 주면 AI는 정답보다 수업에 맞는 설명을 만들기 시작합니다.

HARNESS ENGINEERING

하네스 엔지니어링은
반복 가능한 작업 구조를 만드는 기술입니다.

hand-solution-ai/ input/ problem.md context/ class-profile.md teacher-style.md templates/ solution-template.md checks/ solution-checklist.md output/ solution-draft.md
입력 문제를 읽는다.
수업 컨텍스트와 강사 스타일을 반영한다.
템플릿에 맞춰 해설을 작성한다.
체크리스트로 검증 후 결과를 저장한다.
DIFFERENCE

세 방식은 결과물의
안정성과 반복성에서 차이가 납니다.

기준프롬프트컨텍스트하네스
비유좋은 질문좋은 배경 제공좋은 작업 시스템
결과해설 초안내 학생에게 맞춘 해설반복 생산 가능한 해설 흐름
장점빠르다맞춤성이 좋다일관성과 검증 가능성이 좋다
약점흔들린다맥락 관리가 필요하다초기 설계가 필요하다

좋은 질문에서 좋은 배경 제공으로, 그리고 좋은 작업 시스템으로 확장됩니다.

BLOG AUTOMATION CONTEXT

컨텍스트는 실제로
MD 파일로 관리할 수 있습니다.

# my-class-context.md ## 내 수업의 방향 - 시험 직전 새 문제보다 복습과 점검을 중시 - 학생이 이미 배운 개념을 다시 꺼낼 수 있게 설명 - 불안감을 키우는 표현보다 실천 가능한 행동을 제안 ## 주요 독자 - 고등학교 수학을 준비하는 학생과 학부모 - 시험 직전 학습 방향이 필요한 학생
# blog-output-format.md ## 출력 형식 1. 블로그 제목 3개 2. 본문 구조 - 문제 상황 - 강사의 관점 - 실천 방법 - 마무리 메시지 3. 카드뉴스 문구 6장 4. 인스타 업로드용 요약 문구

AI가 내 생각과 출력 형식을 계속 참고하게 만들면, 콘텐츠 제작도 반복 가능한 흐름이 됩니다.

03. CODEX WORKSPACE

LLM은 답을 만들고,
IDE는 작업공간을 제공합니다.

웹 LLM

대화창 중심

  • 질문하고 답변을 받는다.
  • 결과물을 사람이 복사해서 옮긴다.
  • 아이디어, 초안, 설명에 강하다.
→
IDE / 에이전트

작업공간 중심

  • AI가 폴더 안의 파일을 읽고 수정한다.
  • 실행, 오류 확인, 재수정 루프를 돌린다.
  • 작동하는 결과물 제작에 강하다.

실사용 후기 경향

코드 구현, 리팩토링, 긴 맥락 유지, 프론트엔드 결과물 품질이 좋다는 평가가 자주 언급됩니다.

수업에서의 비유

해설 초안보다 실제 결과물을 깔끔하게 구현해주는 개발 조교.

Gemini / Antigravity

자료 이해와 빠른 시도에 강한 조교

3.1 Pro는 긴 문서·이미지·영상·코드 저장소 분석에, 3.5 Flash는 빠른 속도와 에이전트·코딩 효율에 강점이 있습니다.

실사용 후기 경향

속도, 구글 생태계 연동, 유튜브·문서·이미지 분석 장점이 자주 언급됩니다. 순수 코딩 품질 평가는 갈립니다.

수업에서의 비유

자료를 많이 읽고 빠르게 초안을 여러 번 시도해보는 조사형 조교.

GPT / Codex

전체 작업을 굴리는 PM형 조교

잘하는 일

GPT-5.5는 복잡한 전문 업무·코딩·긴 컨텍스트 작업에 강하고, Codex와 결합해 파일·실행·수정 루프로 연결하기 좋습니다.

실사용 후기 경향

기획, 코딩, 디버깅, 검토까지 이어지는 범용성이 강점으로 언급됩니다. 복잡한 앱은 반복 검증이 필요합니다.

수업에서의 비유

목표를 나누고 작업 흐름 전체를 굴리는 프로젝트 매니저형 조교.

AI 모델 선택은 정답 고르기가 아니라 업무 배치에 가깝습니다. 중요한 것은 어떤 모델을 쓰느냐보다, 목표·자료·검증 기준을 사용자가 얼마나 잘 제공하느냐입니다.
