# 그래프/도형 보조그림 작성 규칙

## 목적

보조그림은 손풀이 감성을 주기 위한 장식이 아니라 풀이 논리를 시각적으로 확인하는 장치이다. 다만 모든 보조그래프가 같은 정확도를 요구하지는 않는다. 그래프가 풀이 결과를 검증하거나 접선, 교점, 좌표값을 직접 보여주는 경우에는 좌표 기반으로 그린다. 반대로 풀이 도중 개형, 증감, 대략적인 위치관계를 떠올리기 위한 경우에는 손풀이 형태의 나이브한 스케치를 허용한다.

## 보조그래프 분류

### A. 좌표 기반 그래프

사용 조건:

- 접선, 교점, 꼭짓점, 대칭축, 특정 좌표가 풀이의 핵심이다.
- 그래프 위 점의 좌표값이 최종 계산에 직접 쓰인다.
- “두 그래프가 접한다”, “두 점에서 만난다”, “이 점을 지난다”처럼 위치 조건의 정확성이 필요하다.
- 검산 또는 정답 확인용 그래프이다.

규칙:

- 실제 좌표계와 좌표 변환을 정한다.
- 함수, 직선, 점, 접점, 교점은 같은 좌표 변환으로 찍는다.
- SVG에는 `data-graph-mode="coordinate"`를 남긴다.
- SVG에는 가능하면 `data-coordinate-system`을 남긴다.
- 손글씨 느낌은 선 색, 라벨, 박스에서 표현하고, 위치는 수학적으로 맞춘다.

### B. 개형 스케치

사용 조건:

- 풀이 도중 “대략 이런 모양”을 보고 추론하는 것이 목적이다.
- 증가/감소, 위로 볼록/아래로 볼록, 축의 방향, 영역의 상대 위치만 필요하다.
- 좌표값이 계산에 직접 쓰이지 않는다.
- 원본 그래프를 해석하기 위한 보조 메모 수준이다.

규칙:

- 손풀이 형태로 나이브하게 표현해도 된다.
- 축, 볼록 방향, 대칭축, 영역의 상대 위치는 문제 조건과 모순되면 안 된다.
- 스케치 라벨에 정확한 좌표처럼 보이는 값을 붙이면 그 값은 실제 식과 맞아야 한다.
- 개형 스케치에는 `개형` 라벨을 기본으로 붙여 정확한 그래프와 구분한다.
- 개형 스케치 라벨은 그림 근처 작은 손글씨 메모로 두고, 풀이 흐름을 방해하지 않게 한다.
- SVG에는 `data-graph-mode="sketch"`를 남긴다.

## 기본 원칙

- 원본 문항의 그래프를 임의로 수정하지 않는다.
- 풀이 여백에 새로 그리는 보조그래프는 먼저 `좌표 기반 그래프`인지 `개형 스케치`인지 분류한다.
- 거리곱 보조그래프는 분류 전에 거리곱을 적용할 대상 함수, 기준선, 목표점, 근/다중도를 먼저 확정한다. 도함수에서 근을 찾았더라도 실제 거리곱 값이 원함수의 높이면 원함수 그래프를 중심으로 그린다.
- 좌표 기반 그래프는 보기 좋게 단순화할 수는 있지만, 접점, 교점, 꼭짓점, 기준점의 상대 위치가 식과 맞아야 한다.
- 개형 스케치는 손풀이 감성을 우선할 수 있지만, 문제 조건과 정반대의 모양을 그리면 안 된다.
- 정확한 그래프가 필요한데 좌표 기반으로 만들기 어렵다면 부정확한 그림을 넣지 않고 식 풀이를 우선한다.

## 좌표 기반 그래프 작성 절차

1. 그래프에 필요한 실제 좌표 범위를 정한다.
   - 예: `x:[-1.2,5.8]`, `y:[-8,8]`
2. SVG 또는 캔버스의 plot 영역을 정한다.
   - 예: `left=65`, `top=32`, `width=300`, `height=276`
3. 실제 좌표를 화면 좌표로 변환한다.

```text
screenX = left + (x - xmin) / (xmax - xmin) * width
screenY = top + (ymax - y) / (ymax - ymin) * height
```

4. 함수 그래프는 여러 실제 `x`값을 대입해 점 목록으로 만든다.
5. 직선, 접선, 보조선은 실제 직선식 위의 두 점을 같은 변환식으로 찍어 그린다.
6. 접점, 교점, 꼭짓점, 주어진 점은 같은 좌표 변환으로 찍는다.

## 접선 그래프 규칙

접선 문제에서는 아래 검사를 반드시 통과해야 한다.

- 접점 좌표가 원래 곡선 위에 있다.
- 접선이 접점과 외부 기준점을 모두 지난다.
- 접선의 기울기가 곡선의 해당 접점 미분계수와 일치한다.
- 보조그림의 선이 곡선을 두 번 가로지르는 것처럼 보이면 실패로 본다.

예: `y=x^2-4x`, `P(3,-7)`에서 그은 접선

```math
y'=2x-4
```

접점 `x=1`이면 점은 `(1,-3)`, 기울기는 `-2`, 접선은

```math
y=-2x-1
```

접점 `x=5`이면 점은 `(5,5)`, 기울기는 `6`, 접선은

```math
y=6x-25
```

두 직선 모두 `P(3,-7)`을 지나야 한다.

## 렌더링 검수 기준

- 보조그래프가 좌표 기반인지 개형 스케치인지 먼저 확인한다.
- 좌표 기반 그래프는 최소한 `source`, `verified_solution_md`, `solution_json`의 핵심 좌표와 일치한다.
- 좌표 기반 SVG에는 `data-graph-mode="coordinate"`와 `data-coordinate-system`처럼 좌표 범위를 남긴다.
- 접선/교점/꼭짓점이 있는 문제는 시각 검수에서 “그림이 풀이식과 같은 조건을 표현하는가”를 확인한다.
- 개형 스케치는 좌표 일치보다 볼록 방향, 축, 상대 위치, 영역 관계가 문제 조건과 모순되지 않는지 확인한다.
- 개형 스케치에는 `개형` 라벨이 있는지 확인한다.
- 개형 스케치 SVG에는 `data-graph-mode="sketch"`가 있는지 확인한다.
- 정확한 그림이 필요한 상황에서 좌표 기반으로 그리기 어렵다면 부정확한 그림을 넣지 않는다.

## 금지

- 접선 문제에서 접선이 아닌 선을 접선처럼 표시하지 않는다.
- 좌표 기반 그래프를 좌표 검증 없이 손으로 대략 그리지 않는다.
- 정확한 접점, 교점 라벨을 임의 위치에 놓지 않는다.
- 그래프가 풀이 결과와 다르면 “개념 설명용”이라는 이유로 유지하지 않는다.
